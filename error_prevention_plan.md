## 2. 코드 개선 계획 (우선순위 순)

### [개선 A] "거래일" 폴백 제거 — 자금집행 완료 필수화 ★★★

**현재 코드:**
```javascript
const primaryTs = dwells["자금집행 완료"]?.enteredTs || null;
const fb1Raw    = d["거래일"];
const fb1       = fb1Raw ? new Date(fb1Raw).getTime() : null;
const fb2       = dwells["결제 완료"]?.enteredTs || null;
const enteredTs = primaryTs || fb1 || fb2 || null;
```

**개선안:**
```javascript
// "자금집행 완료" 단계 진입 날짜만 사용 — 폴백 제거
const enteredTs = dwells["자금집행 완료"]?.enteredTs || null;
```

**효과:** "자금집행 완료" 단계에 진입하지 않은 딜은 집계에서 완전 제외 → 14건 오류 해소  
**부작용 검토:** 과거 딜 중 "결제 완료"/"거래일"만 있는 경우 집계에서 빠질 수 있음. 팀 검토 필요.

---

### [개선 B] 구매이력 미입력 딜 — "미분류" 별도 표시 ★★

**현재 코드:**
```javascript
} else {
  cat = "new_";  // 구매이력 필드 미입력 시 신규로 처리
}
```

**개선안:**
```javascript
} else {
  cat = "unclassified";  // 구매이력 미입력 → 별도 집계
}
```

추가로:
- `row.unclassified = { count:0, orgs:[] }` 항목 추가
- 차트 범례에 "미분류 (구매이력 없음)" 항목 표시 (회색 세그먼트)
- 미분류 건수가 0이 아닐 경우 카드 헤더에 경고 배지 표시: ⚠️ 미분류 N건

---

### [개선 C] Repeat 딜 이전 이력 조회 범위 확장 ★

**현재 문제:** `wonDeals`가 최근 12개월만 포함 → 13개월 이전 거래는 `prevDeals`에서 누락 → 신규로 오분류

**개선안 — 전체 기간 이력 별도 로드:**
```javascript
// wonDeals는 기존대로 최근 12개월
// orgHistory는 전체 기간 딜로 별도 계산
const orgHistory = allDeals
  .map(d => {
    const dwells = parseStageDwells(d, "세일즈 파이프라인");
    const ts = dwells["자금집행 완료"]?.enteredTs || null;
    const orgId = String(d.organizationId || ...);
    return { orgId, ts };
  })
  .filter(d => d.ts)
  .sort((a, b) => a.ts - b.ts);

// Repeat 분류 시 orgHistory 기준으로 이전 이력 검색
const prevDeals = orgHistory.filter(x => x.orgId === d.orgId && x.ts < d.enteredTs);
```

## 3. 운영 가이드라인 (세일즈팀 공유용)

### 필수 입력 필드 정의

| 필드명 | 입력 시점 | 입력 기준 |
|---|---|---|
| **자금집행 완료** (단계 진입) | 실제 자금 집행 확인 시 | 계약서 서명 또는 입금 확인 당일 |
| **구매이력** | 딜 생성 시 | 최초 거래 → New / 재구매 → Repeat |
| **거래일** | (선택) 계약 체결일 기록용 | 자금집행 완료 단계 진입과 동일 날짜 권장 |

### 주의 사항

1. **"거래일" 입력 ≠ 자금집행 완료**: "거래일"만 입력하고 세일즈 파이프라인의 "자금집행 완료" 단계로 이동하지 않으면 대시보드 집계에 영향을 줄 수 있음 (개선 A 적용 후에는 "거래일" 미반영)
2. **구매이력 필드 반드시 입력**: 미입력 시 신규로 자동 분류되어 리텐션/재활성화 수치가 과소 집계됨
3. **진행 중단 딜 처리**: 협의 불발 또는 계약 취소 딜은 즉시 파이프라인에서 "실패" 단계로 이동 처리
